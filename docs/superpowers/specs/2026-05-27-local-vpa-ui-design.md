# 本地量价分析界面设计 Spec

## 1. 背景

当前项目的执行入口是 CLI：`scripts/run_vpa_structure.py` 解析参数后调用
`vpa_structure_recognizer.pipeline.run_pipeline(...)`。核心分析链路已经集中在
Python 包内，并输出项目自有 DuckDB 表和 Excel 报告。

因此，本地界面的第一目标不是重写分析系统，而是把现有能力做成一个可操作、
可浏览、可复核的研究工作台。

## 2. 目标

第一版本地界面要解决五件事：

1. 参数化执行量价结构分析。
2. 浏览本次或历史输出结果。
3. 快速筛选候选池和风险池。
4. 查看单只股票的多窗口结构解释。
5. 允许用户选择全市场判断口径。

成功标准：

- 不通过 shell 拼 CLI 命令，而是直接调用 `run_pipeline(...)`。
- 不改动现有量价判断、评分、标签和导出规则。
- 所有展示结果来自项目生成的 DuckDB 表或 Excel 等价逻辑。
- 同一输入数据和配置应产生可复现的分析结果。
- LLM 不参与结构判断、评分、标签、筛选和数据补全。
- 全市场判断口径必须在本次运行结果中记录，避免不同口径结果混淆。

## 3. 非目标

第一版不做以下内容：

- 不做公网部署、登录系统、权限系统。
- 不做自动交易、交易建议或下单接口。
- 不把分析结果写回上游 DuckDB 数据源。
- 不引入新的量价规则或改动评分算法。
- 不做复杂策略回测平台。
- 不要求接入 LLM；LLM 只作为后续可选解释层。
- 不做完整配置编辑器；第一版只允许选择配置文件和少量运行参数。
- 不新增指数或板块原始数据下载能力；界面只消费上游 DuckDB 已存在的数据。

## 4. 推荐技术方案

推荐使用 Streamlit 构建本地界面。

理由：

- 项目当前是 Python 数据分析管线，Streamlit 接入成本最低。
- 表格、筛选、图表、文件下载和本地执行都能快速覆盖。
- 不需要单独维护前后端协议。
- 适合本地研究工具的迭代节奏。

约束：

- Streamlit 是新增依赖，实施前需要在依赖管理方式上明确记录。
- 界面层不得包含核心分析规则。
- 界面层不得直接散落业务 SQL，复杂查询应封装在服务层函数里。

## 5. 架构设计

建议新增两层：

### 5.1 服务层

建议模块名：`vpa_structure_recognizer/app_service.py`

职责：

- 封装一次分析运行：
  - 参数校验
  - 全市场判断口径校验
  - 调用 `run_pipeline(...)`
  - 返回输出路径、报告路径、表行数、错误信息
- 封装结果读取：
  - 读取 `vpa_structure_state`
  - 读取 `vpa_bar_context_labels`
  - 读取 `vpa_sequence_stats`
  - 读取 `vpa_trend_context`
- 封装业务视图：
  - 全A市场结构
  - 全市场判断口径说明
  - 板块结构排名
  - 强势板块候选
  - 三层共振个股
  - 低位承接增强个股
  - 健康上涨个股
  - 高位供应风险个股
  - 放量破位风险个股
- 封装单股详情：
  - 多窗口状态
  - 评分拆解
  - 风险标记
  - 确认/否定条件
  - 标签明细

服务层是后续替换 UI 技术的稳定边界。

### 5.2 界面层

建议入口：`apps/local_vpa_ui.py`

职责：

- 渲染本地页面。
- 收集用户输入参数。
- 调用服务层函数。
- 展示表格、图表、状态和文件链接。

界面层不直接实现量价判断，不直接重写 Excel 筛选逻辑。

## 6. 页面设计

### 6.1 运行分析页

功能：

- 选择配置文件，默认 `config/default.toml`。
- 选择数据源：
  - `research_source`
  - `audited_stock`
  - 自定义 DuckDB 路径
- 选择全市场判断口径：
  - 自行计算全A宽度和等权OHLC
  - 指定上游宽基指数
  - 混合口径：指数趋势 + 自行计算的市场宽度
- 输入开始日期、结束日期、可选 as-of 日期。
- 输入输出 DuckDB 路径和报告目录，默认使用配置文件值。
- 点击执行后显示：
  - 输出 DuckDB 路径
  - Excel 报告路径
  - 各 `vpa_*` 表行数
  - 执行错误和堆栈摘要

行为约束：

- 执行前校验日期格式、日期顺序、数据源文件是否存在。
- 执行前校验指定指数在上游数据源中存在日线数据。
- 默认输出仍写入 `outputs/`。
- 不覆盖上游源数据库。

### 6.2 全市场判断口径

第一版应把全市场判断口径作为用户可选参数，而不是固定写死。

可选口径：

1. 自行计算全A，作为默认口径。
   - 当前实现已经采用该方式。
   - 从个股日线聚合：
     - 等权 open/high/low/close
     - 总成交额
     - 总成交量
     - 上涨/下跌家数
     - 涨停/跌停家数
     - 20/60 日新高新低数量
     - 强势股/弱势股比例
     - 个股收益率中位数
   - 适合判断市场宽度，不受单一大盘指数权重影响。

2. 指定上游宽基指数。
   - 从上游 `index_daily_bar_pit` 读取指数 K 线。
   - 指数只负责市场趋势和量价结构判断。
   - 如果缺少宽度字段，则不能替代上涨家数、新高新低、强弱股比例等宽度指标。

3. 混合口径，作为推荐增强口径。
   - 指数 K 线用于趋势和价格位置。
   - 自行计算的全A聚合用于市场宽度和风险偏好。
   - 适合同时保留指数趋势和全市场参与度。

当前默认研究源中可用于“指定上游宽基指数”的日线指数：

| 指数代码 | 名称 | 数据范围 | 说明 |
| --- | --- | --- | --- |
| `000906.SH` | 中证800 | 20140102 至 20260525 | 有 `index_daily_bar_pit` 日线 |
| `000985.CSI` | 中证全指 | 20140102 至 20260525 | 有 `index_daily_bar_pit` 日线，更接近全市场口径 |
| `399317.SZ` | 国证A指 | 20140102 至 20260525 | 有 `index_daily_bar_pit` 日线 |

`index_basic_ref` 还列出 `700001.TI`（同花顺全A(加权)），但当前默认研究源的
`index_daily_bar_pit` 没有对应日线记录。界面可以展示为“引用存在但无日线”，
不能作为可执行口径，除非上游补齐指数日线。

运行结果需要记录本次市场口径，例如：

```text
market_mode = self_calculated_all_a
market_index_code = null
market_scope_id = ALL_A
```

或：

```text
market_mode = selected_index
market_index_code = 000985.CSI
market_scope_id = 000985.CSI
```

### 6.3 结果总览页

功能：

- 选择一个输出 DuckDB 文件。
- 展示本次全市场判断口径。
- 展示最近分析日期。
- 展示全A市场结构状态。
- 展示评级分布：A/B/C/D。
- 展示结构状态分布。
- 展示板块评分 Top N。
- 展示风险标记数量。

数据来源：

- `vpa_structure_state`

### 6.4 候选池页

功能：

- 以 tab 或选择器展示现有报告口径：
  - 三层共振个股
  - 低位承接增强个股
  - 健康上涨个股
  - 高位供应风险个股
  - 放量破位风险个股
  - 个股结构总表
- 支持筛选：
  - 日期
  - 评级
  - 结构状态
  - 板块
  - 风险标记
  - 置信度区间
- 支持按评分、置信度、成交额、换手率等字段排序。
- 支持导出当前筛选结果为 CSV。

数据口径：

- 与 `excel_exporter.py` 中已有 sheet 逻辑保持一致。

### 6.5 个股详情页

功能：

- 输入或选择股票代码。
- 展示：
  - 最终评级
  - 最终结构状态
  - 趋势背景
  - 位置背景
  - 市场/板块/个股/共振评分
  - 多窗口状态：10/20/30/60/120/240
  - 主特征
  - 风险标记
  - 看强确认条件
  - 看弱否定条件
- 展示该股多窗口标签明细。
- 展示该股序列统计。
- 生成一段确定性中文解释。

解释来源：

- 第一版使用模板化解释，复用或扩展 `report_formatter.py`。
- 模板解释只翻译已有字段，不新增判断。

### 6.6 报告与导出页

功能：

- 展示最近生成的 Excel 报告路径。
- 提供本地下载或打开路径提示。
- 展示已生成的输出文件列表。
- 支持对当前筛选表导出 CSV。

## 7. 板块数据口径

当前实现里的板块不是直接读取 Tushare 或上游板块指数 K 线。

当前板块生成方式：

- `ResearchSourceDuckDB` 从上游 `industry_classification_pit` 读取个股所属行业。
- `build_sector_bars(...)` 按 `industry_code` 和 `industry_name` 分组。
- 每个板块的 open/high/low/close 使用板块内成分股均值。
- volume 和 amount 使用板块内成分股合计。
- advancers/decliners、limit_up/limit_down、member_count 从成分股统计。

这属于项目本地自行计算的行业板块聚合，不是 Tushare 板块指数 K 线。

后续如果上游提供正式板块指数 K 线，界面和服务层应支持板块口径选择：

1. 自行计算行业板块，作为当前默认。
2. 上游板块指数 K 线。
3. 混合口径：板块指数趋势 + 成分股内部宽度。

第一版界面至少应展示当前板块口径，避免用户误以为板块结构来自外部板块指数。

## 8. LLM 边界

第一版不需要 LLM。

如果后续接入 LLM，只允许作为可选、只读解释层：

- 输入：结构状态、评分、标签、风险标记、确认/否定条件等已有字段。
- 输出：更自然的中文说明、候选池总结、问答解释。
- 禁止：修改评分、修改标签、补造行情、生成交易指令、改变筛选结果。

推荐策略：

- 默认显示模板解释。
- 可选按钮：“生成自然语言解读”。
- LLM 输出必须标注为“基于结构字段的文字解释”，不能标注为新的分析结论。

## 9. 错误处理

界面需要处理以下错误：

- 配置文件不存在或 TOML 解析失败。
- 数据源 DuckDB 文件不存在。
- 指定指数在 `index_daily_bar_pit` 中不存在。
- 指定指数在所选日期范围内没有日线数据。
- 开始日期晚于结束日期。
- 上游数据缺少必要字段。
- 输出路径不可写。
- 分析过程中出现异常。
- 输出 DuckDB 文件不存在或表缺失。

错误展示原则：

- 给用户显示简短中文错误。
- 保留开发者可用的异常摘要。
- 不吞掉异常，不显示“成功”状态。

## 10. 测试要求

新增测试应优先覆盖服务层，而不是 Streamlit 渲染细节。

建议测试：

- 参数校验：日期、路径、source alias。
- 全市场口径校验：自行计算、指定指数、无日线指数、日期范围无数据。
- 运行服务：使用现有 smoke fixture 或临时 DuckDB，确认返回 `PipelineResult` 等价信息。
- 结果读取：能从输出 DuckDB 读取各核心表。
- 候选池筛选：与 Excel sheet 逻辑一致。
- 板块口径展示：当前默认口径明确显示为本地行业聚合。
- 个股详情：缺失代码、存在代码、多窗口字段缺失时行为明确。
- 模板解释：只使用传入字段，不产生额外判断。

验证命令：

```bash
python -m pytest tests -v
```

如新增 UI 依赖，还需要至少执行一次本地启动 smoke：

```bash
streamlit run apps/local_vpa_ui.py
```

## 11. 实施顺序

建议分五步实施：

1. 新增服务层和测试，包括市场口径发现与校验。
2. 扩展管线参数，支持自行计算全A、指定指数、混合口径三种市场判断方式。
3. 新增 Streamlit 本地界面。
4. 增加个股模板解释和导出体验。
5. 后续在上游提供正式板块指数 K 线后，再接入板块指数口径。

每一步都应保持现有 CLI 可用。

## 12. 验收标准

第一版完成后，应满足：

- CLI 仍可运行。
- 本地界面可以启动。
- 用户可以从界面执行一次分析。
- 用户可以选择全市场判断口径。
- 指定指数时，界面只允许选择有日线数据的指数。
- 当前板块口径被明确标注为本地行业聚合。
- 用户可以选择输出 DuckDB 并浏览结果。
- 候选池页面与 Excel 报告筛选口径一致。
- 个股详情页能解释一只股票的结构状态。
- 全量测试通过。
- 没有引入 LLM 到核心判断链路。
